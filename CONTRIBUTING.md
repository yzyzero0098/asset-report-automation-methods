# Contributing

This repository is a sanitized automation template. Contributions should keep
the project source-neutral and safe for public reuse.

## Good Contributions

- improve generic adapter interfaces
- add synthetic fixtures
- add validation or dry-run checks
- clarify scheduling or reproducibility steps
- improve sanitization checks

## Please Do Not Add

- real provider names
- real issuer names
- real target domains, login pages, or admin paths
- credentials, cookies, tokens, or session details
- production PDFs, spreadsheets, or logs

## Pull Request Checklist

1. Use placeholders such as `SOURCE_A` and `ASSET_001`.
2. Keep examples synthetic.
3. Confirm scripts default to dry-run behavior.
4. Run the sanitization scan from `docs/sanitization_policy.md`.
5. Explain what part of the template the change improves.
